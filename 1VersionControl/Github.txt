* Git restore command:
- Iss command ko use karke hum restore kar sakte h jo hamne last commit kiya tha
- suppose hum commit kar chuke hai xyz and abhi humne uss file me xyzb kara hai par hume ab previous comit code wapis chahiye matlab file me xyz wapis chahiye
- toh hum git restore filename.extension ya git restore . kar skte hai

git restore .
(basically kisi file ko track se untracked me lane ke liye isko use kiya hai)
(git add ke baad jo file tracked me gayi h usko wapis restore ki ja skti hai na ki git commit ke baad ... commit ke baad restore nhi kiya ja skta hai)

- git restore undo changes in working directory
- it does not effect history
- usually local use or undo local file changes
- works on files
- no new commit create here

* Git Revert command:
- undo a commit in history
- it affects the history as it creates new commit
- safe for shared repos
- works on commits
- new commit create here

git revert <commit-hash>
(each commit has unique hash value)

* Git reset command:
- it works on commits/HEAD
- It changes history sometimes
- not safe for pushed code
- no new commit create here
- move branch pointer to previous commit
- it is risky

Soft reset:

git reset --soft HEAD^
(^ means previous commit. moves head, keeps staging and keeps working files)

Hard reset:

git reset --hard HEAD^
(^ means previous commit. moves head, delete staging and delete working files)

- git reset moves your branch back to an older commit
- but it can also:
- keep your changes - this is called a soft reset
- unstage your changes - this is called a mixed reset
- or delete your changes - this is called a hard reset

Suppose your commits look like this:
A → B → C → D (HEAD)

if we run : git reset HEAD~1

then our commits will look like this:
A → B → C (HEAD)

means commit D is removed from history

* Git Branches:
A branch is just a seperate line of development

Without branches:
A → B → C → D

With branches:
A → B → C → D (main)
         \
          E → F (feature branch)

- See all the branches:
git branch
(show local branches and show current branch as *)

- Create a new branch:
git branch new_branch_name
(create new branch but we are still on main)

- Switch to a branch:
git checkout new_branch_name
OR 
git switch new_branch_name
(now we are working in new_branch_name)

- Create a new branch and switch to it:
git checkout -b new_branch_name
OR 
git switch -b new_branch_name
(now we are working in new_branch_name => creates branch and switches immediately)

- Delete a branch:
git branch -d new_branch_name

- Rename a branch:
git branch -m old_branch_name new_branch_name

- Merge a branch:
First go to main:

git switch main

then merge

git merge new_branch_name
(new_branch_name will be merged into main)

* Git pull and fetch command:
- git fetch : download (safe)
git fetch origin
(It downloads latest changes from remote, does not change your code and does not merge)
(It only updates remote tracking branches)

- git pull : download + merge (automatic)
git pull origin main
(git fetch + git merge)

Before pull:
local main:      A → B
origin/main:     A → B → C → D

After pull:
local main:      A → B → C → D
origin/main:     A → B → C → D


* Git Diff command:
- git diff shows what changed in your code

Before (last commit):
const name = "Pulkit";

After (edited):
const name = "Pulkit Gupta";

Run: git diff

You will get:
- const name = "Pulkit";
+ const name = "Pulkit Gupta";

1. git diff (working vs staged)
(show unstaged changes)

2. git diff --staged (or --cached)
(show staged vs last commit)

3. git diff HEAD^ (or HEAD~1)
(show last commit vs previous commit eg. git diff a1b2c3 d4e5f6)

4. git diff between branches:
(git diff main feature-login : show difference between main and feature-login)

5. git diff for a specific file:
(git diff filename.extension)

* Git stash command:
- Git stash ka matlab apne current changes ko temporary side me rakh do
- Jab aapka kaam complete nhi hai but branch siwtch karni hai - stash use hota hai

- Maan lo:
- Tum feature par kaam kar rahe ho
- Kaam complete nahi
- Suddenly bug fix karna hai dusri branch me

❌ commit nahi karna chahte
❌ changes lose nahi karna chahte

👉 git stash = life saver

1. changes stash karne ke liye
git stash
(current changes safe ho jate hain, working directory clean ho jati haim HEAD last commit par aa jata hai)

2. Stash list dekhne ke liye
git stash list

3. Stash wapas lana
git stash apply
(isse changes wapaas aa jate hai par stash list me uska entry remove nhi hota)

4. Apply + delete (clean way)
git stash pop

5. Clear all stashes:
git stash clear
